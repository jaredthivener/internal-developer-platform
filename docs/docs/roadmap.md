# Roadmap

## Near Term

- Continue replacing Crossplane-first language with application-first concepts.
- Improve the main dashboard so environments summarize compute, storage, networking, and data services together.
- Expand the public documentation set with screenshots and onboarding material.
- Persist intended resource state outside the management cluster before Crossplane apply.
- Add Azure-backed resource observation and drift classification for storage accounts.

## Mid Term

- Add operational workflows for upgrades, rollbacks, and patching.
- Introduce clearer environment health and deployment status views.
- Document authentication and authorization flows for enterprise deployments.
- Add sync-back remediation through Crossplane for supported resource types.
- Validate Velero-based control-plane recovery and adoption fallback for dynamic resources.

## Documentation Work

Documentation is now a first-class deliverable for the repository. The minimum expected coverage is:

- Architecture overview
- Local development setup
- Deployment and hosting model
- Product concepts and terminology
- Roadmap and operating assumptions
